## 一、项目概述
### 1. 项目背景
随着城市公共服务设施不断增多，居民和外来人员在日常出行、临时休憩、旅游打卡、就医救助、亲子照护等场景中，对便民设施空间信息的获取需求日益增强。现有公共设施信息分散在不同平台，存在查询入口分散、信息更新不及时、服务对象不够精细化等问题。学生、游客、宝妈、老年人、车主等人群需求差异明显，传统统一式地图服务难以满足个性化需求。

同时，地理信息服务正从“地图展示”向“基于场景的个性化空间服务”发展，不同用户关注重点不同：游客关注打卡点、公厕、停车点；宝妈关注母婴室、医疗设施；老年人关注养老、就医设施；车主关注停车与充电设施。因此，构建基于身份角色分类、提供差异化地图服务的便民服务网页，具有现实意义与课程实践价值。

### 2. 项目目标
本项目设计并实现基于WebGIS的多角色城市便民服务平台，整合充电桩、母婴室、公厕、停车点、养老设施、医疗设施、旅游打卡点等公共设施信息，面向不同用户提供个性化图层推荐与地图服务，支持设施状态上报与应急求助查询，提升便民信息服务的针对性、动态性与实用性。

具体目标：
1. 构建多类型便民设施空间数据库；
2. 实现地图浏览、分类查询、点位详情展示等基础GIS服务；
3. 设计个性化身份角色首页，按用户需求自动推荐图层；
4. 实现设施状态上报功能，增强信息更新能力；
5. 拓展应急求助与公益救助服务模块，提升社会服务价值。

## 二、需求分析
平台面向**学生、游客、亲子家庭、老年人、车主/骑手、应急求助者**等用户，不同人群需求差异显著：
- 学生：手机充电、公厕、医疗点、临时休息点
- 游客：旅游打卡点、公厕、停车点、拍照引导、周边服务
- 亲子家庭：母婴室、公厕、医疗设施、亲子休憩空间
- 老年人：养老设施、医疗设施、无障碍设施、休息点
- 车主/骑手：充电桩、停车点、临时补给点
- 应急求助者：临时住宿、公益餐食、救助站、警务站、急救服务

核心需求：
1. 展示多类便民设施点位，支持分类图层切换；
2. 基于用户身份自动推荐对应设施图层；
3. 提供点位详情查询、关键词搜索、距离筛选；
4. 支持用户上报设施故障、关闭、位置错误等状态；
5. 提供应急求助与公益救助信息查询；
6. 界面简洁，适配移动端与电脑端。

## 三、研究方案
### 1. 研究内容
1. 构建城市便民设施空间数据库，整合多类空间点位数据，统一字段与坐标；
2. 设计多角色个性化首页，按身份自动推荐图层与服务卡片；
3. 实现基础WebGIS功能：地图加载、图层控制、点位检索、属性弹窗、类别筛选、周边查询；
4. 实现设施状态上报功能，支持运行状态、开放情况、位置信息反馈；
5. 拓展应急求助与公益救助模块，整合临时住宿、公益餐食、救助站等资源。

### 2. 技术路线
1. 收集整理便民设施数据，完成坐标统一、字段标准化、去重分类；
2. 构建便民服务空间数据库，按设施类型与适用人群建立属性字段；
3. 采用WebGIS技术实现底图加载、图层控制、点位显示、空间查询；
4. 开发角色选择模块，按身份自动加载推荐图层；
5. 开发设施上报、应急求助模块；
6. 界面优化、交互测试、功能完善，形成完整WebGIS网页。

**技术栈**：
- 前端：HTML + CSS + JavaScript
- 地图：高德地图JS API / Leaflet
- 数据：GeoJSON / JSON / CSV
- 上报：前端模拟提交
